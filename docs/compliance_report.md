## 一、审计方法论

```
1. 全量文件扫描 → 2. 关键词检测 → 3. 逐文件审查 → 4. 依赖链分析 → 5. 结论输出
```

### 1.2 检测关键词

| 类别 | 关键词 |
|------|--------|
| 版权声明 | `Copyright`, `License`, `MIT`, `GPL`, `BSD`, `Apache` |
| 引用标记 | `borrowed from`, `adapted from`, `based on`, `derived from`, `ported from` |
| 外部链接 | `github.com`（非自身仓库）, `gitlab.com`, `bitbucket.org` |
| 代码指纹 | 已知 Linux/BSD 内核代码片段哈希 |

### 1.3 审计工具

- 全文本搜索: 对所有 `.c`, `.h`, `.S`, `.py`, `.sh`, `.md`, `.yml`, `Makefile` 执行关键词匹配

- 依赖扫描: 分析 Makefile、CI/CD 配置中的外部工具依赖
