### 1.4 内存分配流程

```
   用户/内核请求内存
         │
         ▼
   ┌─────────────┐
   │  kmalloc()  │
   │  size ≤ 4KB?│
   └──┬──────┬───┘
      │ 是   │ 否 (>4KB)
      ▼      ▼
   ┌──────────┐   ┌──────────────────┐
   │  Slab    │   │  alloc_pages()   │
   │  分配器   │   │  Buddy 伙伴系统   │
   └────┬─────┘   └────────┬─────────┘
        │                  │
        ▼                  ▼
   ┌──────────────────────────────────────┐
   │         Slab 缓存查找流程              │
   │                                      │
   │  size=32B  → cache[0]  32B  slab    │
   │  size=64B  → cache[1]  64B  slab    │
   │  size=128B → cache[2]  128B slab    │
   │  size=256B → cache[3]  256B slab    │
   │  size=512B → cache[4]  512B slab    │
   │  size=1KB  → cache[5]  1KB  slab    │
   │  size=2KB  → cache[6]  2KB  slab    │
   │  size=4KB  → cache[7]  4KB  slab    │
   │                                      │
   │  命中 → 返回空闲对象                   │
   │  未命中 → alloc_pages(0) 获取新页     │
   │           → 初始化为 slab 缓存        │
   └──────────────────────────────────────┘
        │
        ▼
   ┌──────────────────────────────────────┐
   │         Buddy 伙伴系统                │
   │                                      │
   │  order=10 (1024页, 4MB)  ◄── 最大   │
   │  order=9  (512页,  2MB)             │
   │  order=8  (256页,  1MB)             │
   │  ...                                │
   │  order=1  (2页,    8KB)             │
   │  order=0  (1页,    4KB)   ◄── 最小  │
   │                                      │
   │  分配: 从对应 order 链表取页          │
   │  不足: 向上分裂高 order 块            │
   │  释放: 检查伙伴是否空闲 → 合并        │
   └──────────────────────────────────────┘
        │
        ▼
   ┌──────────┐
   │  物理页   │
   │  (4KB)   │
   └──────────┘
```

### 2.3 模块间数据流 (调用链)

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                            模块间调用链 (Call Chain)                          │
└─────────────────────────────────────────────────────────────────────────────┘

  【文件读取路径】
  user_app → syscall → sys_read → vfs_read → ext2_read → block_dev_read → ramdisk_read

  【文件写入路径】
  user_app → syscall → sys_write → vfs_write → ramfs_write / ext2_write → block_dev_write

  【进程创建路径】
  shell → syscall → sys_fork → fork() → clone_current_pml4() → COW 页表复制
       → syscall → sys_execve → elf_load() → map_user_page() → 创建用户页表

  【信号递送路径】
  sender → syscall → sys_kill → do_sys_kill → 设置 pending_signals
       → 返回用户态前 → signal_deliver → signal_setup_frame → 用户 handler

  【网络数据路径】
  user_app → syscall → sys_read/write → netdev → virtio_net → virtqueue → 网络

  【模块加载路径】
  shell → mod load → module_load → vfs_open → elf 解析 → module_register_symbol

  【性能监控路径】
  shell → perf 命令 → perf_dump → 遍历 perf.counters[] → 输出到控制台
              ← perf_inc() 分散在各模块中 (调度器/系统调用/缺页/内存分配/中断)
```
